# 🚀 Minha Jornada QA - Semana 1

## 🧠 O que aprendi sobre Mindset
* **QA (Quality Assurance):** Garantir a qualidade e prevenir erros.
* **Diferenças fundamentais:**
    * **Erro:** Humano.
    * **Defeito (Bug):** No código.
    * **Falha:** No uso real.

## 📝 Documentação de Testes
### Casos de Teste (CT)
Devem ter ID, Título, Passos e Resultado Esperado.

### Reporte de Bug
Deve ser claro e ter:
1. Título direto.
2. Passos para reproduzir.
3. Resultado atual vs. Esperado.
4. Evidência (Print/Vídeo).

## ⚖️ Severidade vs Prioridade
* **Severidade:** O quão "partido" está o sistema (Técnico).
* **Prioridade:** O quão rápido temos de arranjar (Negócio).

## 🔄 Processos (Scrum)
* **Sprint:** Ciclo de 2 semanas de trabalho.
* **Daily:** Reunião diária de 15 min.
